Payments

Keep both rails. Let the client choose.

Stripe handles familiar invoice payments. x402 gives an HTTP payment-required path for crypto-native clients.

Stripe

Card and invoice payment support with webhook-ready backend handling.

x402

USDC-oriented payment confirmation path for Base Sepolia workflows.

Reputation

Paid invoices create portable work records for future trust flows.

Deploy

Live backend. Static frontend. Render-safe.

Backend runs at hermeswork.onrender.com. Frontend calls the backend directly and supports browser-stored API keys for protected writes.

GET /health → ok
GET /api/kpis → live dashboard data
POST /demo/seed → API-key protected sample data
POST /invoice/create → real backend write
FAQ

Practical answers.

Is the dashboard connected?

Yes. The frontend now defaults to https://hermeswork.onrender.com on Render and only uses localhost during local development.

Why do I need an API key?

Write actions are protected. Save HERMESWORK_API_KEY in Settings, then create invoices or seed sample data.

What is real today?

The backend, read APIs, protected write APIs, Stripe integration path, x402 payment route, and reputation record flow are implemented.
